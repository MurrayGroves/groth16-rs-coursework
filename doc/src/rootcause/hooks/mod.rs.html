1//! Hooks system for customizing report creation and formatting behavior.
2//!
3//! # When to Use Hooks
4//!
5//! **Most users don't need hooks** - the defaults work well. Use hooks when you
6//! need to:
7//! - Automatically attach data to ALL errors (request IDs, timestamps,
8//!   environment info)
9//! - Integrate with custom logging or observability systems
10//! - Change how reports are formatted globally (different colors, layout,
11//!   structure)
12//! - Redact or transform sensitive data in error messages
13//!
14//! **If you just need to customize a single error**, use `.attach()` or
15//! handlers (see [`examples/custom_handler.rs`]) instead of hooks.
16//!
17//! [`examples/custom_handler.rs`]: https://github.com/rootcause-rs/rootcause/blob/main/examples/custom_handler.rs
18//!
19//! # Modules
20//!
21//! - **[`report_creation`]**: Automatically add data to every report as it's
22//!   created (e.g., request IDs, correlation IDs, environment variables)
23//!
24//! - **[`formatting_overrides`]**: Control how specific types appear in error
25//!   messages (e.g., redact passwords, format timestamps, control attachment
26//!   placement)
27//!
28//! - **[`report_formatting`]**: Change the entire report layout and structure
29//!   (e.g., JSON output for logging, compact format, custom colors)
30//!
31//! - **[`builtin_hooks`]**: Default hooks that are automatically registered
32//!   (location collectors, backtrace collectors, and the default formatter)
33//!
34//! See [`examples/report_creation_hook.rs`] and
35//! [`examples/formatting_hooks.rs`] for complete examples.
36//!
37//! [`examples/report_creation_hook.rs`]: https://github.com/rootcause-rs/rootcause/blob/main/examples/report_creation_hook.rs
38//! [`examples/formatting_hooks.rs`]: https://github.com/rootcause-rs/rootcause/blob/main/examples/formatting_hooks.rs
