1//! # WARNING
2//! This library was made for purely academic purposes, and has not been audited for security!
3//! Don't use this in production!
4//!# Groth16
5//!
6//!This crate provides a library for generating and verifying Groth16 proofs over a given Rank 1 Constraint System.
7//!
8//!The implementation is type generic over any curve that implements `ark_ec::Pairing`, but is tested in this library with
9//!MNT6-753 since it's the highest security MNT curve.
10//!
11//!The public interface is well-documented and may be viewed at `docs/groth16/index.html`, or by running
12//!`cargo doc --open`.
13//!
14//!Additionally, errors are handled using the `rootcause` crate which allows for developer friendly backtraces including
15//!additional context and attached debug information (think showing operands to a failed call). All Results are annotated
16//!with at the minimum a `.context()` call describing the operation that failed.
17//!
18//!The core types are `circuits::R1CS`, `circuits::QAP`, `groth16::TrustedSetupOutput`, `groth16::Proof`.
19//!
20//!The general flow is:
21//!
22//!- Define R1CS
23//!- Use `QAP::from` to convert R1CS to QAP
24//!- Generate a Trusted Setup using `TrustedSetupOutput::new`
25//!- Generate a proof using `trusted_setup.prove(witness)`
26//!- Verify proof with `proof.verify()`
27//!
28//!The library is extensively tested with 100% test coverage, which can be verified with
29//!`cargo install tarpaulin; cargo tarpaulin --engine llvm` or by looking at the coverage report in `coverage`. Note that
30//!although the coverage report reports 414/418 lines covered, the missing 4 lines are all match arms that are incorrectly
31//!marked as being code, which can be verified by going into `polynomial.rs` in the report and finding the red lines.
32//!
